## Prerequisites

Before installing ReactFlix, ensure your development environment meets these requirements:

### Required Software

<AccordionGroup>
  <Accordion title="Node.js (v14 or higher)">
    ReactFlix requires Node.js for running the development server and building the application.
    
    **Check your version:**
    ```bash
    node --version
    ```
    
    **Install Node.js:**
    - Download from [nodejs.org](https://nodejs.org/)
    - Use nvm (Node Version Manager): `nvm install 18`
    - Recommended: Node.js v18 LTS or higher
  </Accordion>

  <Accordion title="npm (v6 or higher)">
    npm is the package manager for JavaScript and comes bundled with Node.js.
    
    **Check your version:**
    ```bash
    npm --version
    ```
    
    **Update npm:**
    ```bash
    npm install -g npm@latest
    ```
  </Accordion>

  <Accordion title="Git (Optional but Recommended)">
    Git is needed to clone the repository.
    
    **Check your version:**
    ```bash
    git --version
    ```
    
    **Install Git:**
    - Download from [git-scm.com](https://git-scm.com/)
    - macOS: `brew install git`
    - Ubuntu: `sudo apt-get install git`
  </Accordion>
</AccordionGroup>

### System Requirements

- **Operating System**: Windows 10+, macOS 10.14+, or Linux
- **RAM**: Minimum 4GB (8GB recommended)
- **Disk Space**: 500MB for node_modules and build files
- **Browser**: Modern browser (Chrome 90+, Firefox 88+, Safari 14+, Edge 90+)

## Configuration

### Package.json Scripts

ReactFlix includes several npm scripts for different tasks:

```json package.json
{
  "scripts": {
    "start": "react-scripts start",
    "build": "react-scripts build",
    "test": "react-scripts test",
    "eject": "react-scripts eject"
  }
}
```

<CodeGroup>

```bash Development Mode
npm start

# Starts development server
# Opens browser at http://localhost:3000
# Hot reloading enabled
# Shows lint errors in console
```

```bash Production Build
npm run build

# Creates optimized production build
# Output directory: build/
# Minified and optimized
# Includes hashed filenames for caching
```

```bash Run Tests
npm test

# Launches Jest test runner
# Interactive watch mode
# Runs tests on file changes
# Press 'a' to run all tests
```

```bash Eject (Advanced)
npm run eject

# ⚠️ One-way operation - cannot undo!
# Exposes webpack config
# Full control over build process
# Only use if you need customization
```

</CodeGroup>

### Environment Variables

Create a `.env` file in the root directory for custom configuration:

```bash .env
# Server Port (default: 3000)
PORT=3000

# Browser auto-open (default: true)
BROWSER=true

# Enable HTTPS in development
HTTPS=false

# Advanced webpack config (if ejected)
GENERATE_SOURCEMAP=true
```

<Note>
  Environment variables must start with `REACT_APP_` to be accessible in your code (except built-in variables like PORT).
</Note>

## Troubleshooting

### Common Installation Issues

<AccordionGroup>
  <Accordion title="npm install fails">
    **Problem**: Errors during `npm install`
    
    **Solutions**:
    ```bash
    # Clear npm cache
    npm cache clean --force
    
    # Delete node_modules and reinstall
    rm -rf node_modules package-lock.json
    npm install
    
    # Try using npm ci for clean install
    npm ci
    ```
  </Accordion>

  <Accordion title="Port 3000 already in use">
    **Problem**: "Port 3000 is already in use"
    
    **Solutions**:
    ```bash
    # Find and kill process on port 3000
    # macOS/Linux:
    lsof -ti:3000 | xargs kill -9
    
    # Windows:
    netstat -ano | findstr :3000
    taskkill /PID <PID> /F
    
    # Or use different port
    PORT=3001 npm start
    ```
  </Accordion>

  <Accordion title="Module not found errors">
    **Problem**: "Module not found: Can't resolve..."
    
    **Solutions**:
    ```bash
    # Ensure you're in the correct directory
    pwd  # Should show path ending in /streaming
    
    # Reinstall dependencies
    npm install
    
    # Check for typos in import statements
    # Verify file paths are correct
    ```
  </Accordion>

  <Accordion title="npm start shows blank page">
    **Problem**: Browser shows blank white page
    
    **Solutions**:
    1. Open browser DevTools (F12)
    2. Check Console for errors
    3. Verify all files are present in src/
    4. Check that public/index.html exists
    5. Clear browser cache and hard reload (Ctrl+Shift+R)
    
    ```bash
    # Reset and restart
    rm -rf node_modules build
    npm install
    npm start
    ```
  </Accordion>

  <Accordion title="Node version incompatibility">
    **Problem**: "The engine 'node' is incompatible"
    
    **Solutions**:
    ```bash
    # Check your Node version
    node --version
    
    # Upgrade Node.js to v14+
    # Using nvm:
    nvm install 18
    nvm use 18
    
    # Then reinstall dependencies
    npm install
    ```
  </Accordion>
</AccordionGroup>
