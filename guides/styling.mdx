ReactFlix uses a traditional CSS approach with well-organized stylesheets and BEM (Block Element Modifier) naming conventions. This guide covers the styling architecture, patterns, and customization options.

## Styling Architecture

ReactFlix uses three main CSS files for organization:

<Steps>
  <Step title="App.css - Global Styles">
    **Location**: `src/styles/App.css`
    
    **Purpose**: Global resets, body styles, and app-level layout
    
    **Size**: 31 lines
    
    ```css
    * {
      margin: 0;
      padding: 0;

    }

    body {
      font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', 'Roboto';

      flex-direction: column;
    }

    .app__main {
      flex: 1;

      margin: 0 auto;
    ```

  </Step>
  
  <Step title="components.css - Component Styles">
    **Location**: `src/styles/components.css`
    
    **Purpose**: Styles for all reusable components
    
    **Size**: 411 lines
    
    **Includes**:
    - Header navigation

    - Movie cards

    **Size**: 257 lines
    
    **Includes**:

    - Catalog page layout
    - Detail page grid
    - Management pages
    - Responsive breakpoints
  </Step>
</Steps>

### Import Order

Styles are imported in `src/App.js:10` in a specific order:

```javascript
import './styles/App.css';        // Global styles first
import './styles/components.css'; // Components second
import './styles/pages.css';      // Pages last
```

<Note>
This import order ensures proper CSS cascade and specificity management.
</Note>

## BEM Methodology

ReactFlix follows BEM (Block Element Modifier) naming convention throughout:

### BEM Structure

<Accordion title="Block">
**Definition**: Standalone component with independent meaning

**Pattern**: `.block-name`

**Example**:
```css
.header { }
.pelicula-card { }
.search-bar { }
```
</Accordion>

<Accordion title="Element">
**Definition**: Part of a block with no standalone meaning

**Pattern**: `.block-name__element-name`

**Example**:
```css
.header__logo { }
.header__nav { }
.header__nav-list { }
.pelicula-card__image { }
.pelicula-card__title { }
```
</Accordion>

<Accordion title="Modifier">
**Definition**: Different state or version of a block/element

**Pattern**: `.block-name--modifier-name`

**Example**:
```css
.header__nav-link--active { }
.button--disabled { }
.modal--open { }
```
</Accordion>

## Interactive States

### Hover Effects

ReactFlix implements smooth hover transitions:

<Accordion title="Card Hover">
```css

  transition: transform 0.3s;

.pelicula-card:hover {
```

## Customization Guide

### Changing Colors

To change the brand color, update these values:

```css
/* Primary brand color */
.header__logo { color: #YOUR_COLOR; }
.header__nav-link--active { color: #YOUR_COLOR; }
.footer__title { color: #YOUR_COLOR; }
.pelicula-card__trailer-btn { background-color: #YOUR_COLOR; }
.principal-page__title { color: #YOUR_COLOR; }
/* ... and more */
```

<Tip>
Use CSS custom properties (variables) for easier theming:

```css
:root {
  --brand-color: #e50914;
  --bg-dark: #141414;
  --bg-card: #1f1f1f;
}

.header__logo {
  color: var(--brand-color);
}
```
</Tip>

### Adjusting Spacing

```css
/* Main content padding */
.app__main {
  padding: 20px; /* Change to 40px for more space */
}

/* Grid gap */
.pelicula-grid {
  gap: 20px; /* Change to 30px for wider gaps */
}
```

### Modifying Typography

```css
/* Change font family */
body {
  font-family: 'Your Font', sans-serif;
}

/* Adjust sizes */
.principal-page__title {
  font-size: 48px; /* Change to 64px for larger hero */
}
```
